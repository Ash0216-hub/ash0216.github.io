HTTP / 웹 통신

HTTP Request Response

TCP / UDP 차이

TCP는 연결 지향적이며 신뢰성을 보장하고, UDP는 연결 과정이 단순하며 빠르지만 별도의 확인 절차가 필요하다는 점을 학습하였습니다.

TCP UDP Socket

DNS / 도메인

도메인 이름과 IP 주소의 관계, 그리고 GitHub Pages와 같은 웹 호스팅 환경에서 링크를 통해 홈페이지를 배포하는 과정을 이해하였습니다.

DNS Domain Hosting

프로젝트 소개

이번 미니프로젝트에서는 브라우저 기반 인터페이스와 내부 TFTP 동작을 결합한 파일 전송 시스템을 구현하였습니다.

TFTP 미니프로젝트 개요

본 프로젝트는 사용자가 웹브라우저를 통해 파일을 업로드하거나 다운로드할 수 있도록 구성하였으며, 프로그램 내부에서는 UDP 기반 TFTP 방식으로 파일 전송이 이루어지도록 설계하였습니다.

주요 구현 기능

  • 브라우저 기반 파일 업로드
  • 서버 파일 목록 조회
  • 파일 다운로드
  • UDP 기반 RRQ / WRQ / DATA / ACK 처리
  • loopback 환경(127.0.0.1) 테스트

사용 기술

본 프로젝트는 C++ 기반으로 구현하였으며, HTTP 서버 처리와 UDP 소켓 기반 TFTP 로직을 함께 구성하였습니다.

C++ HTTP UDP TFTP

업로드 동작

브라우저에서 업로드 요청이 들어오면 내부적으로 WRQ를 전송하고, 이후 DATA와 ACK를 주고받으며 파일을 저장하도록 구현하였습니다.

WRQ DATA ACK

다운로드 동작

브라우저에서 다운로드를 요청하면 내부적으로 RRQ를 전송하고, 서버로부터 받은 DATA 블록에 ACK를 응답하며 파일을 수신하도록 구현하였습니다.

RRQ DATA ACK
