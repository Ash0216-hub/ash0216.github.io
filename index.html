Core Concepts

수업에서 학습한 네트워크 개념을 전기공학적 관점과 함께 정리했습니다.

HTTP와 웹 통신

브라우저와 서버가 요청과 응답을 주고받는 구조를 학습했습니다. 사용자가 화면에서 보는 결과가 실제로는 통신 절차를 통해 전달된다는 점을 이해할 수 있었습니다.

TCP와 UDP

TCP는 신뢰성을 보장하는 통신 방식이고, UDP는 구조가 단순하고 빠르다는 특징이 있습니다. 전송 방식의 차이를 비교하면서 시스템 설계에서의 선택 기준을 생각해볼 수 있었습니다.

DNS와 주소 체계

사람이 읽기 쉬운 도메인 이름이 실제 IP 주소와 연결되는 구조를 이해했습니다. 이를 통해 인터넷에서 서비스가 식별되고 접근되는 과정을 배울 수 있었습니다.

통신 시스템 관점

이번 학습은 단순한 웹페이지 제작을 넘어서, 데이터가 송신되고 수신되는 절차와 프로토콜 흐름을 이해하는 과정이기도 했습니다. 전기공학에서 배우는 통신 시스템 개념과 연결해서 볼 수 있었습니다.

System Design

TFTP 기반 파일 전송 프로젝트를 시스템 설계 관점에서 요약했습니다.

프로젝트 목표

브라우저 기반 인터페이스와 UDP 기반 파일 전송 구조를 연결하여, 사용자가 업로드와 다운로드를 수행할 수 있는 흐름을 이해하는 것이 목표였습니다.

Goal

동작 구조

사용자는 웹페이지에서 요청을 보내고, 내부적으로는 RRQ, WRQ, DATA, ACK와 같은 TFTP 패킷 흐름이 동작하는 구조를 정리했습니다.

Flow

구현 의의

HTTP와 UDP 기반 통신이 어떤 역할을 하는지 구분해서 볼 수 있었고, 통신 프로토콜이 실제 시스템 구현으로 이어지는 과정을 경험할 수 있었습니다.
