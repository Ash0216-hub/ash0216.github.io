About Me

저를 간단히 소개하는 공간입니다.

저는 전기공학과 학생으로서 하드웨어뿐 아니라 소프트웨어와 네트워크 시스템에도 관심을 가지고 있습니다. 이번 컴퓨터네트워크 과제를 통해 단순히 이론을 배우는 것에서 끝나지 않고, 실제로 웹페이지를 만들고 GitHub Pages로 배포하는 경험까지 해볼 수 있었습니다.

수업에서 배운 내용

이번 수업에서 특히 인상 깊었던 개념들을 정리했습니다.

웹과 HTTP

브라우저가 서버에 요청을 보내고, 서버가 응답을 돌려주는 구조를 학습했습니다. 또한 웹페이지가 실제로 인터넷에 배포되는 과정도 직접 경험했습니다.

TCP와 UDP

TCP는 신뢰성 있는 통신을 위한 방식이고, UDP는 빠르고 단순한 통신 방식이라는 점을 비교하며 배웠습니다. TFTP가 UDP 기반으로 동작한다는 점도 함께 이해할 수 있었습니다.

DNS와 도메인

사람이 이해하기 쉬운 도메인 이름이 IP 주소와 어떻게 연결되는지, 그리고 GitHub Pages 링크가 어떻게 동작하는지 알게 되었습니다.

배포 경험

HTML 파일을 작성하고 GitHub 저장소에 올린 뒤, Pages 기능으로 실제 링크를 생성해 인터넷에서 접속 가능한 홈페이지를 만드는 과정을 실습했습니다.

프로젝트 정리

TFTP 기반 파일 전송 프로젝트를 간단히 요약했습니다.

프로젝트 목표

브라우저를 통해 파일을 업로드하거나 다운로드하고, 내부적으로는 TFTP 방식으로 파일이 전송되도록 구현하는 것이 목표였습니다.

주요 기능

파일 업로드, 파일 다운로드, 서버 내 파일 목록 조회, 그리고 RRQ / WRQ / DATA / ACK 흐름을 이해하고 정리했습니다.

사용 기술

C++, HTTP, UDP, TFTP와 같은 기술을 기반으로 네트워크 통신이 실제로 어떻게 이루어지는지 확인했습니다.

느낀 점

이번 과제를 하며 느낀 점을 정리했습니다.

처음에는 홈페이지 제작과 배포 과정이 어렵게 느껴졌지만, 직접 하나씩 따라 하면서 HTML 파일이 실제 웹페이지가 되고, GitHub Pages를 통해 링크로 공개된다는 점이 인상 깊었습니다. 이번 과제를 통해 네트워크 개념뿐 아니라 간단한 웹 배포 과정까지 경험할 수 있었습니다.
