Core Concepts

수업에서 학습한 주요 개념을 전기공학적 시각과 함께 정리했습니다.

HTTP와 웹 서비스 구조

브라우저와 서버가 요청과 응답을 주고받는 구조를 이해하면서, 사용자가 화면에서 보는 결과가 실제로는 여러 통신 절차 위에서 구성된다는 점을 배울 수 있었습니다.

TCP와 UDP의 전송 특성

TCP는 신뢰성과 순서를 보장하는 방식이고, UDP는 구조가 단순하고 빠르다는 특성을 가집니다. 각 프로토콜의 차이를 비교하면서 상황에 따라 적절한 전송 방식이 달라진다는 점을 이해했습니다.

DNS와 인터넷 주소 체계

사람이 읽기 쉬운 도메인 이름이 실제 IP 주소와 연결되는 과정을 학습하며, 인터넷에서 서비스가 어떻게 식별되고 접근되는지 구조적으로 이해할 수 있었습니다.

통신 이론과 시스템 구현의 연결

이번 과제를 통해 네트워크 이론은 단순한 개념 암기가 아니라, 실제 시스템 설계와 동작 절차를 이해하기 위한 기반이라는 점을 체감할 수 있었습니다.

System Design

TFTP 기반 파일 전송 프로젝트를 설계와 구현의 관점에서 정리했습니다.

프로젝트 목표

브라우저 기반 사용자 인터페이스와 UDP 기반 파일 전송 구조를 연결하여, 파일 업로드와 다운로드가 어떤 통신 흐름 위에서 수행되는지 이해하는 것을 목표로 했습니다.

Goal

동작 흐름 분석

사용자의 요청이 웹페이지에서 시작되어 내부적으로는 RRQ, WRQ, DATA, ACK와 같은 TFTP 패킷 교환으로 이어지는 구조를 정리하며 전체 시스템 흐름을 파악했습니다.

Flow

구현의 의미

이번 구현을 통해 HTTP와 UDP 기반 통신의 역할 차이를 구분해볼 수 있었고, 프로토콜이 실제 동작하는 시스템 안에서 어떤 방식으로 사용되는지 더 명확하게 이해할 수 있었습니다.

Meaning

Insight & Reflection

이번 과제를 통해 얻은 배움과 인상을 정리했습니다.

이번 과제는 단순히 결과물을 제출하는 작업이라기보다, 전기공학에서 배우는 통신 개념이 실제 시스템 구현과 어떻게 맞닿아 있는지 확인하는 과정에 가까웠습니다. 처음에는 HTML 작성과 GitHub Pages 배포가 낯설고 다소 막막하게 느껴졌지만, 파일 하나가 브라우저에서 동작하는 웹페이지가 되고, 그 결과가 다시 인터넷을 통해 하나의 서비스처럼 공개되는 경험은 매우 인상적이었습니다. 또한 TFTP 기반 파일 전송 구조를 정리하면서 데이터의 이동이 단순한 파일 복사가 아니라 프로토콜, 패킷 흐름, 응답 절차로 구성된 정교한 과정임을 이해할 수 있었습니다. 이번 경험을 통해 저는 이론을 배우는 것에 머무르지 않고, 실제 동작 원리를 구조적으로 바라보는 시각과 시스템 단위로 사고하는 태도의 중요성을 더 분명하게 느끼게 되었습니다.
