About Me

저를 간단히 소개하는 공간입니다.

저는 전기공학과 학생으로서 회로와 신호 분야뿐 아니라, 네트워크와 소프트웨어 시스템에도 관심을 가지고 있습니다. 이번 과제를 통해 HTML 파일을 직접 작성하고, GitHub Pages로 실제 웹페이지를 배포하는 과정을 경험했습니다. 단순히 이론을 배우는 것에서 그치지 않고, 결과물을 인터넷 링크 형태로 공개해볼 수 있었다는 점이 인상적이었습니다.

학습 내용

수업에서 배운 주요 개념을 정리했습니다.

웹과 HTTP

브라우저와 서버가 요청과 응답을 주고받는 구조를 배우면서, 사용자가 보는 웹페이지가 어떤 방식으로 전달되는지 이해할 수 있었습니다.

TCP와 UDP

TCP는 신뢰성 중심, UDP는 빠르고 단순한 통신 방식이라는 차이를 비교하며 각각이 어떤 상황에 적합한지 학습했습니다.

DNS와 도메인

사람이 읽기 쉬운 도메인 이름이 실제 IP 주소와 연결된다는 점, 그리고 GitHub Pages 주소가 어떻게 동작하는지도 함께 이해했습니다.

웹 배포 경험

HTML 파일을 작성하고 GitHub 저장소에 올린 뒤, Pages 기능을 통해 실제 접속 가능한 홈페이지를 배포하는 과정을 직접 수행했습니다.

프로젝트 정리

TFTP 기반 파일 전송 프로젝트를 간단히 요약했습니다.

프로젝트 목표

브라우저를 통해 파일을 업로드하고 다운로드할 수 있도록 구성하고, 내부적으로는 TFTP 방식의 파일 전송 흐름을 이해하는 것이 목표였습니다.

주요 기능

파일 업로드, 다운로드, 파일 목록 확인, 그리고 RRQ, WRQ, DATA, ACK와 같은 패킷 흐름을 학습하고 정리했습니다.

사용 기술

C++, HTTP, UDP, TFTP 등 다양한 기술 요소를 바탕으로 네트워크 통신의 실제 동작 방식을 이해할 수 있었습니다.

느낀 점

이번 과제를 하며 느낀 점을 정리했습니다.

처음에는 홈페이지 제작과 배포 과정이 낯설고 어렵게 느껴졌지만, 직접 HTML 파일을 만들고 GitHub Pages로 연결해 보면서 하나의 파일이 실제 웹페이지가 된다는 점을 체감할 수 있었습니다. 이번 과제를 통해 네트워크 개념뿐 아니라 간단한 웹 배포 과정까지 함께 경험할 수 있어 의미 있었습니다.
